# 愿景

一个人，一群 Agent，一个调度台。

飞书机器人只是起点。最终我想要的是一个自己用的 Web 界面——可以跑在局域网里，也可以只在桌面上——打开它就像打开一个指挥中心。里面没有人类同事，但有一群各怀本事的 AI Agent 等着接活。

我是唯一的用户，也是唯一的指挥官。

## 两种打法

**指哪打哪**。我有一个项目，列了 5 件要做的事。我看一眼手里的 Agent 阵容——Claude Code 擅长深度推理和复杂重构，Codex 出活快适合批量任务，还有我自己造的一些专精 Agent，比如对接公司内部 API 的、专门做日志分析的。我把任务一件件分下去，谁合适就给谁。干完了我看结果，不满意打回去重来。

**组队协作**。有些任务一个 Agent 搞不定，或者搞得慢。我把几个 Agent 拉进同一个项目，扔一个任务进去，它们自己根据各自的能力分工。快但粗的先跑一版，慢但精的做精修，专精某个领域的在旁边辅助。我不用操心谁做哪部分，只管看最终交付的东西够不够好。

## Agent 不只是现成的

Claude Code 和 Codex 只是开始。未来我要能自己造 Agent——对接自己的 API，绑定特定的工具链，训练在某个狭窄领域里做到极致。这些 Agent 像乐高积木，随时可以被拉进任何一个项目的团队里。

有的 Agent 是全能选手，什么都能干一点；有的是专精型，只做一件事但做得极好。一个好的团队两种都需要。

## 为什么要做这个

因为一个人的时间和精力是有限的，但想做的事情是无限的。如果我能带着一群 Agent 像带团队一样工作——分配任务、协调分工、把控质量——那一个人能做到的事情就不再受限于一个人的产出。

这不只是我想要的，也是我相信自己能做到的。这是终极的远景——可能不会一步到位，但方向不会变。
